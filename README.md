# 古诗文搜索

面向课堂演示、朗读辅助与内容整理的古诗文检索与排版工具。输入题目和作者后，服务端会聚合本地知识库与在线补全结果，输出适合阅读、批注、导出和展示的完整页面。

## 项目特点

- 多源检索：优先使用课本知识库与归档数据，缺失内容再走在线补全，减少空结果和错误匹配。
- 结构化排版：正文支持逐字拼音、译文、注释、朗诵参考与同页阅读。
- 跨端阅读：针对手机、平板和桌面分别优化布局、触控目标、批注滚动、长图导出与 PDF 预览。
- 科学背诵：提供熟读、渐隐、无提示三阶段训练，支持语音、默写、自评、逐字纠错、薄弱句与进度恢复。
- 个人诗库：最近阅读、收藏和背诵进度保存在当前浏览器，可从首页快速继续。
- 易于部署：纯 Python 服务，可直接本地运行，也可通过 Docker 部署到 Render、Railway 等平台。

## 适用场景

- 课堂展示或诵读教学
- 诗词全文查询与整理
- 带拼音的阅读页生成
- 需要快速分享给他人的轻量站点

## 快速启动

> 这个项目不是纯静态网页。前端会请求 `server.py` 提供的接口，因此需要按 Web 服务方式运行，而不是只上传 `index.html`。

```powershell
python -m pip install -r requirements.txt
python server.py
```

默认访问地址：

```text
http://127.0.0.1:8765
```

## 运行结构

- `server.py`：HTTP 服务入口，负责查询、补全、导出和静态资源分发
- `lookup_classical_text.py`：内置古诗文检索脚本，避免依赖本机外部技能目录
- `textbook_knowledge_base.json`：课本知识库
- `top500_knowledge_base.json`：归档诗词数据
- `app.js` / `reader.html`：排版阅读页、批注模式与导出交互
- `recite-scroll.js` / `recite-scroll.html`：渐隐背诵、语音/默写检查、薄弱句与本地进度
- `tests/test_recitation.py`：背诵差异、同音识别与静态资源安全测试

## 背诵训练说明

1. 先用“熟读”完整看一遍正文。
2. 切到“渐隐”，根据首字和少量线索主动回忆。
3. 最后用“无提示”检查是否真正掌握。
4. 语音识别出现同音字时不会直接判错或静默放行，而会提示确认；手动默写始终按字形严格检查。
5. “提示一字”只揭示当前句的一个字，“核对本句”用于语音不可用时的自评，不再一次显示整页答案。

收藏、最近阅读和背诵进度使用版本化 `localStorage`，默认不保存录音。当前版本没有账号体系，因此这些数据不会自动跨设备同步。

## 部署方式

### Docker

项目已自带 `Dockerfile`，适合直接接入云平台构建。

```powershell
docker build -t poem-search .
docker run --rm -p 8765:8765 poem-search
```

### Render / Railway

1. 在平台创建新的 Web Service。
2. 连接此 GitHub 仓库。
3. 让平台直接使用仓库中的 `Dockerfile` 构建。
4. 部署完成后即可通过平台分配的公网域名访问。

### 临时分享

如果只是短期演示，也可以在本机启动服务后通过隧道工具暴露端口：

```powershell
python server.py
cloudflared tunnel --url http://127.0.0.1:8765
```

或使用 `ngrok` 等同类工具。此方式依赖本机持续在线。

## 环境变量

| 变量名 | 说明 |
| --- | --- |
| `PORT` | 云平台注入的监听端口，服务会自动读取 |
| `POEM_UI_PORT` | 本地自定义端口，默认 `8765` |
| `POEM_UI_HOST` | 监听地址，云部署建议使用 `0.0.0.0` |
| `POEM_DATA_DIR` | 缓存、导出文件与运行时数据目录 |
| `POEM_HUMAN_VERIFICATION_TTL_SECONDS` | 人机验证有效期，默认 21600 秒 |
